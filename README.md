# VAE-approach-to-1D-Spike-Sorting
This repository contains the code and methodology for evaluating the utility of Variational Autoencoder (VAE) augmented data in training neural networks to resolve complex, overlapping extracellular spikes. 

Rather than focusing on general spike sorting, this project isolates the specific algorithmic bottleneck of multi-spike collisions.

## Repository Structure

* `VAE_approach.ipynb`: The primary Jupyter Notebook containing the training loops, model definitions, and multi-run statistical evaluation.
* `data_generation.py`: Scripts for generating synthetic datasets, VAE augmentations, and linear superpositions.
* `models.py`: Definitions for the baseline and VAE-augmented 1D Convolutional Neural Network architectures.
* `evaluation.py`: Custom evaluation callbacks and stratified metric calculations.
* `visualization.py`: Utilities for plotting waveforms, training histories, and performance metrics.
* `requirements.txt`: Dependency list for replicating the environment.

## Methodology

To objectively measure the impact of synthetic training data on loss landscapes and gradient stability, this project benchmarks three distinct 1D Convolutional Neural Network (CNN) training regimens:

* **Model A (Starved Baseline):** Trained exclusively on pure, isolated single spikes.
* **Model B (VAE-Augmented):** Trained on highly dense, multi-spike collisions generated and reconstructed via a Variational Autoencoder latent space.
* **Model C (Linear Superposition):** Trained on collisions generated by the simple linear addition of natural spikes, preserving biological morphology but compounding thermal noise.

Every model undergoes a statistical multi-run evaluation ($N=3$) to account for variations in random weight initialization and gradient trajectories.

## Conclusion & Future Work

Resolving extracellular collisions is limited by the scarcity of multi-spike ground-truth data. I compared three approaches: a baseline (Model A), VAE-augmented synthetic data (Model B), and linear superposition (Model C). While all models resolved collisions, results reveal critical differences in training dynamics and stability.

### 1. Performance and Stability
Model C (linear superposition) demonstrated high training stability, likely because it preserves the original spike morphology and thermal noise characteristics. Model B (VAE-augmented) achieved a performance ceiling comparable to Model C but exhibited significant variance across runs. While VAE-generated data effectively synthesizes overlaps, it lacks the natural noise regularization found in Model C, leading to a more sensitive loss surface.

### 2. Gradient Dynamics
Model B’s instability is attributed to gradient domination. Because VAE data is pristine and morphologically complex, it creates high loss penalties early in training. If the initial weights are sub-optimal, the optimizer struggles to converge. Model C avoids this by using linear mixtures, which provide a more consistent gradient path throughout training.

### 3. Future Work
* **Curriculum Learning:** To stabilize the VAE approach, training should begin exclusively with isolated spikes. Once the network anchors its filters, the synthetic overlaps can be introduced gradually to prevent divergence.
* **Temporal Bottlenecks:** Current 1D segmentation fails for spikes occurring within 5 samples, as their probability frames merge into a single plateau. To surpass this limit, future algorithmic pipelines should integrate multi-channel data from high-density electrode probes (e.g., Neuropixels) to decouple these overlaps across physical space.
